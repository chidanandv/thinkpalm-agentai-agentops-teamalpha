      Fleet Health — User Guide

      Multi-agent pipeline for maritime fleet health & delivery reporting

      What is this?

      This Operations Center ingests maritime operational data — daily noon reports, port schedules, bunker logs, and maintenance alerts — and produces a Fleet Health & Delivery Report for ship management teams. Four specialised AI agents run in sequence via LangGraph, with results stored in SQLite for history and traceability.

      Key features

      • 4-agent pipeline — Ingestion, Anomaly Detection, Performance Summary, Escalation
      • Dashboard — Health score, posture bar, KPIs, charts, trends, vessel cards, filters
      • Export & print — Download JSON or print the current report
      • Theme toggle — Switch between dark and light mode (🌙 in header)
      • Priority alerts — Sidebar feed for critical/high items; click to jump to details
      • Report comparison — Delta banner shows anomaly/escalation change vs previous run
      • Export CSV — Download anomalies table for spreadsheets
      • Keyboard shortcuts — R run pipeline, F refresh, ? guide
      • Report history — Previous runs saved and reloadable from the sidebar
      • REST API — Generate reports programmatically; OpenAPI docs at /docs
      • Hybrid intelligence — Rule-based anomaly thresholds plus Claude narratives when an API key is set
      • Deterministic mode — Runs without Claude for CI and local demos

      How to use the dashboard

      1. Start the server — Run start.bat or python -m fleet_health (default port 8001).
      2. Open http://127.0.0.1:8001/ — The latest saved report loads automatically if history exists.
      3. Run sample pipeline — Click the sidebar button to process bundled Fleet Alpha sample data through all four agents (~2–5 seconds).
      4. Explore tabs — Overview (executive summary), Vessels, Anomalies, Escalations, and Agent trace.
      5. Load history — Pick a past report from the dropdown and click Load selected report.
      6. API Docs — Use the top-bar button to test endpoints interactively in Swagger UI.

      How the pipeline works

      Noon reports → Ingestion → Anomaly → Performance → Escalation → Report
      • Ingestion — Parses and normalises voyage, port, bunker, and maintenance inputs; saves vessel snapshots.
      • Anomaly detection — Flags fuel overconsumption (>10%), schedule slippage (>6h), and overdue maintenance.
      • Performance — Assigns Green / Amber / Red status per vessel with fuel, schedule, and maintenance posture.
      • Escalation — Surfaces critical defects needing shore-side superintendent action.

      Data inputs also include port call schedules, bunker consumption logs, and PMS maintenance alerts.
